# Architecture Decision Records (ADR)

Dossier où documenter chaque décision architecturale significative.

## Format recommandé

Nommage : `NNNN-titre-court.md` (ex: `0001-choix-monorepo-pnpm.md`).

Structure minimale d'un ADR :

```markdown
# NNNN — Titre de la décision

Date : YYYY-MM-DD
Statut : proposée | acceptée | remplacée par NNNN | obsolète

## Contexte
Pourquoi cette décision se pose.

## Options envisagées
- Option A — pour/contre
- Option B — pour/contre

## Décision
Ce qu'on a retenu et pourquoi.

## Conséquences
Ce que ça implique (positif et négatif).
```
